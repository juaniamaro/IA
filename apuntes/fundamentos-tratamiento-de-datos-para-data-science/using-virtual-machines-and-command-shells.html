COMMAND SHELL AND VIRTUAL MACHINES

A shell is, by definition, an application that acts as an interface between the user and the operating system. Its main function is to interpret commands written in text, execute them, and return the results also in text format. In addition to processing simple commands, the shell allows you to automate tasks using scripts, manage processes, redirect input and output, chain commands, and control the working environment (variables, permissions, paths, etc.). Thanks to this, it becomes a very powerful tool for both daily use and system administration and development. In short, the shell not only allows you to communicate with the operating system, but also offers a complete control language that facilitates advanced interaction, automation, and efficient management of system resources.

VIRTUAL BOX AND THE CONCEPT OF VIRTUAL MACHINES

The existence of different operating systems creates the disadvantage that we have to create different versions of the same program to adapt it to them, but we don't do it many times because it is not profitable for companies. Virtual machines work by using an operating system installed on the host system that simulates another system.

!
To create and run a virtual machine, special software is required to perform the simulation. Some of the most popular options are VirtualBox and VMware.

When using virtual machines, the following should be taken into account:

1 2 3 4 5
All operations performed on and from the virtualized operating system will not affect the host operating system or other guest operating systems. Each time a virtual machine is run, at the moment it is shut down, it is possible to save the state of the virtual machine. Document exchange between the host and guest operating systems is possible. Auxiliary resources such as network access, printers, or other devices can be used. Each virtual machine is allocated resources according to those available from the host operating system.

Among the cloud services offered by companies that specialize in them is the creation of virtual machines, also called instances. The instance is given a series of characteristics and the instance is associated with a public IP that will be used for the connection.

CREATING AND CONFIGURING A VIRTUAL MACHINE

INSTALLATION REQUIREMENTS

  • RAM Memory: Minimum 8 GB
  • Storage: At least 20 GB of free space
  • Processor: A relatively modern processor with x86 architecture

INSTALLATION

2.CREATION AND CONFIGURATION OG VIRTUAL MACHINE

For create a virtual machine, first you need the image of the system that you want emulate

Lubuntu Download Page

Step 1

Once the system image is downloaded, we open VB and press [button name] to give it a type and version.

Step 2

You must choose how much memory to allocate to the virtualized system.

Step 3

In the next window, verify that the "Create virtual disk now" option has been selected and click on Create.

Step 4

In the next window, you choose the type of file you want to use for the virtual hard drive.

Step 5

The type of storage on the hard drive is selected.

Step 6

The user is asked where the file should be saved. By default, it will be saved in: “C:\Users\username\VirtualBox VMs\Virtual-Machine-Name”. This can be changed by clicking on the folder icon. The maximum hard drive size can also be configured in this window.

Step 7

Clicking on Create opens the updated Virtual Box window with the configured machine.
